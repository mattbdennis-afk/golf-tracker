# Golf Performance Dashboard

A single-file HTML golf performance dashboard hosted on GitHub Pages at **mattbdennis-afk.github.io/golf-tracker**.

Built and maintained with Claude (Anthropic). Data sourced from Garmin Golf / Garmin Connect.

-----

## What the Dashboard Does

Tracks Matt’s golf performance across all rounds since 2020. Key features:

- **Season view** — KPIs, score trend, strokes gained, benchmark vs 9 HCP goal, round-by-round table, par 3/4/5 averages
- **Rounds view** — Detailed breakdown of any individual round
- **Courses view** — Performance by course
- **Clubs view** — Distance data, GIR%, FIR%, distance gapping analysis (powered by Garmin sensor data)
- **Insights view** — September deep-dive, scoring by month, course performance table, strengths & focus areas
- **Add Data tab** — Round template and instructions for adding new data

-----

## File Structure

Everything lives in a single file: `index.html`

- All round data is stored in the `ROUNDS` array near the top of the script block
- Club data is stored in `CLUB_DATA_BY_YEAR` (auto-computed from Garmin JSON, regenerated by Claude)
- No external dependencies except Google Fonts (optional) — works offline

## How to Add New Rounds

### Option A — Garmin Export (recommended, captures full shot data)

1. In Garmin Connect or Garmin Golf app, export your round data as JSON
1. You get 4 files: `Golf-SCORECARD.json`, `Golf-SHOT.json`, `Golf-COURSE.json`, `Golf-CLUB.json`
1. Start a new Claude session at claude.ai
1. Upload all 4 JSON files plus the current `index.html`
1. Use this prompt:

> “Here are my updated Garmin Golf JSON files. Please merge any new rounds into my existing index.html and regenerate it with all new data included. Keep all existing rounds and the existing dashboard design intact.”

1. Claude will produce a new `index.html` — use the downloader page method (see below) to get the file safely

### Option B — Manual entry

Open `index.html` in a text editor, find the `ROUNDS` array, and paste a new round object using the template in the Add Data tab of the dashboard.

## How to Upload to GitHub (iPad)

**Do not copy-paste the HTML code** — this corrupts special characters and breaks the file.

Instead:

1. Ask Claude to produce a **downloader page** (a small HTML file that contains the dashboard embedded inside it)
1. Open the downloader page from Claude → tap **Download index.html** → file saves to iPad Downloads folder
1. Go to GitHub repo → **Add file → Upload files** → select the downloaded file from Files app → commit

This keeps the file byte-perfect with no encoding corruption.

## Important Technical Notes for Claude

- **Single HTML file** — all CSS, JS, and data in one file. No build process, no separate files.
- **Safari on iPad** — the file must avoid:
  - Template literals (backtick strings) that contain `</tag>` before the closing backtick — Safari misparses the `/` as a regex
  - Non-ASCII characters in JS string literals (em dashes, smart quotes, box-drawing chars) in certain line positions
  - The fix: use plain string concatenation (`h += '<div>...</div>'`) for all HTML-building in JS functions
- **Existing functions** use template literals and non-ASCII safely — do not rewrite them unless necessary
- **New functions** should use string concatenation only (see `renderInsights` and `renderAddData` as examples)
- The `notes()`, `kpi()`, `fillBar()`, `goalColor()`, `fmt()`, `calcSA()`, `calcSeasonSG()`, `roundsForYear()` helper functions are all available globally
- Color constants are in the `C` object: `C.pos`, `C.neg`, `C.warn`, `C.gold`, `C.blue`, `C.dim`, `C.ink`, `C.greenPale`, `C.goldPale`, `C.redPale`
- Goal benchmarks are in the `GOAL` object

## Round Data Format

```javascript
{
  id: "YYYY-MM-DD_CourseName",
  date: "YYYY-MM-DD",
  location: "Course Name",
  tees: "White",
  rating: 70.0,
  slope: 125,
  yards: 6400,
  par: 72,
  roundType: "18hole",   // "18hole" | "9hole" | "exec" — omit for 18hole
  which9: "Front 9",     // only for 9hole rounds
  score: 85,
  adjScore: 85,
  diff: null,
  hcp: 12.0,
  fir: 57.1,             // fairways in regulation %
  playable: 71.4,        // playable fairway % (hit + manageable misses)
  missTeeLeft: 14.3,
  missTeeRight: 14.3,
  ob: 0.0,
  gir: 33.3,
  greenShort: 25.0, greenLong: 12.5, greenLeft: 25.0, greenRight: 37.5,
  approach: {">200":null,"175-200":50,"150-175":40,"125-150":35,
             "100-125":45,"75-100":60,"50-75":70,"<50":75},
  putts: 34,
  onePutt: 16.7, twoPutt: 72.2, threePutt: 11.1,
  girsHit: 6, girPutts: 10,
  sandSaves: "1/2", sandSavePct: 50.0,
  upDowns: "3/6", upDownPct: 50.0,
  penalties: 1,
  par3: 3.50, par4: 4.40, par5: 5.20,
  birdies: 0, pars: 7, bogeys: 8, doubles: 3,
  notes: ""
}
```

Null values are fine — the dashboard handles missing fields gracefully. The most important fields for full functionality are: `date`, `location`, `par`, `score`, `fir`, `playable`, `gir`, `putts`, `threePutt`, `sandSavePct`, `upDownPct`, `penalties`.

## Handicap Tracking

Active handicap is calculated from rounds played **April 1 – October 31** only (Michigan season). Displayed on the Season view. Goal is 9 HCP.
